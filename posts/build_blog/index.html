在 windows 安裝 WSL

啟用 Windows 內的 WSL 設定

在 [開始] 搜尋 windows powershell，並按右鍵以系統管理員身分執行，輸入以下指令：

Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

2021 更新 WSL2

原本 windows 的 WSL 為 version1，近期改版成 version2，接下來教學如何升級到 WSL2。經過一整天的踩雷，最後重新安裝了另一個版本的 linux distro (Ubuntu-20.04)，因為原本的 Ubuntu distro 不知為何就是無法升級成 WSL2，總之筆者用了 wsl --unregister Ubuntu 的指令解除了原本的 distro，然後重新安裝。接下來就是參考官方的教學一步一步安裝。 安裝教學、讓WSL介面更好看

另外，在安裝好 zsh 以及 oh-my-zsh 後，會發現 ls 所產生出來的顏色巨雞巴醜，解決的方法是設定 LS_COLORS，作法：到這裡選一個喜歡的主題下載，例如：

curl https://raw.githubusercontent.com/seebi/dircolors-solarized/master/dircolors.ansi-dark --output ~/.dircolors

接著在 ~./zshrc 裡面加上這行 (參考資料)：

## set colors for LS_COLORS
eval `dircolors ~/.dircolors`

安裝 linux distro

到 Microsoft Store 選擇適合的 WSL 散發套件，筆者選擇安裝 Ubuntu

安裝完成後開啟 App，第一次進入會需要設定使用者名稱以及密碼。除了使用 Ubuntu App 進入外，也可以開啟 Windows 的命令提示字元，使用 bash 或 wsl 進入。Windows 本機的磁碟資料都會存放在 Linux 系統的 mnt 資料夾內。 使用指令 cd /mnt 進入資料夾。

安裝 zsh 以及 oh-my-zsh 套件

此步驟非必要。如果希望終端機可以好看一點，可以考慮安裝。如果不安裝可以直接跳至安裝 jekyll 步驟。安裝完效果如下：

(一) Windows 安裝 Powerline 字體

對於 Windows 用戶而言，必須使用 Powerline 的字體才能顯示出特定的 icon，因此必須到 github Page，找到喜歡的字體的 ttf 檔，下載後在 Windows上安裝，不要裝在WSL的系統，因為WSL是用windows終端機的字體。

:::info [2020-2-11] 根據筆者的測試，似乎不是所有 Powerline 字體都可以正常顯示 icon，經測試安裝完可以正常顯示的是 DejaVu Sans Mono for Powerline.ttf。安裝完後，在之後的步驟如果發現還是無法正常顯示 icon，請使用此方式安裝：How to install Powerline fonts in Windows ::: :::info [2020-2-11] 如果安裝完字體卻發現 Ubuntu 的終端機沒有該字體可以選擇，也許是 windows 的命令提示字元的字集為 950，但我們要使用的字集為 65001，解決方法：Windows 命令提示字元 (CMD) 設定等寬字型 :::

(二) 安裝 zsh

詳細安裝 zsh 以及 oh-my-zsh 步驟請參考：(WSL) 環境設定

下載 zsh

$ sudo apt-get install zsh

參考指令：

  • 檢查版本： $ zsh --version
  • 將 zsh 設為預設 shell： $ chsh -s $(which zsh)
  • 還原成原本預設的 shell： chsh -s /bin/bash
  • 刪除 zsh：sudo apt-get --purge remove zsh

(三) 安裝 oh-my-zsh

$ sh -c "$(wget https://raw.github.com/robbyrussell/oh-my-zsh/master/tools/install.sh -O -)"

安裝完後可進入 vim ~/.zshrc 設定喜歡的主題
